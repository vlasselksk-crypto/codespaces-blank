SlothAC FastAPI API Server - work summary

Created files and contents:

1) requirements.txt
   - fastapi, uvicorn[standard], flatbuffers, python-multipart, pydantic, pytest, httpx, pandas, scikit-learn, joblib, torch, torchvision, numpy

2) README.md
   - instructions to run, test, docker build/run, API endpoints description (updated for LSTM)

3) Dockerfile
   - python 3.12-slim base, installs requirements, copies full project into /app, runs uvicorn

4) docker-compose.yml
   - service definition for slothac-api with port 8000 and env key

5) app/flatbuffers/schema/tickdata.fbs
   - FlatBuffers schema for TickData and TickDataSequence (8 floats per tick)

6) app/main.py
   - FastAPI app with /v1/inference POST (now uses trained LSTM model or 0.1)
   - /train-lstm POST endpoint for uploading CSV files and training LSTM neural network
   - Model persistence: loads model.pth and scaler.pkl on startup, saves after training
   - LSTM architecture: 2 LSTM layers (8->128->64), dropout 0.3, sigmoid output
   - Sequence training: windows of 40 ticks, step 20
   - Data normalization with StandardScaler
   - Early stopping (patience 5), validation split 20%
   - API key auth via X-API-Key header
   - CORS middleware
   - request logging middleware
   - FlatBuffers parsing + error handling

7) app/config.py
   - get_api_key() reads SLOTHAC_API_KEY or defaults to test-key

8) app/flatbuffers/slothac/TickData.py + TickDataSequence.py
   - FlatBuffers python bindings (generated) updated for runtime

9) app/flatbuffers/parser.py
   - parse_tickdata_sequence(bytes) -> list[list[float]]
   - build_tickdata_sequence(rows) -> bytes (for tests)

10) tests/test_api.py
   - TestClient-based tests for auth, parsing, invalid payload, perf
   - New tests: /train-lstm endpoint, inference with trained LSTM model
   - Updated fixtures to remove model.pth and scaler.pkl

11) work_done.txt (this file)

12) model.pth + scaler.pkl
   - Trained LSTM model and scaler (from tests)

Commands run:
- pip install -r requirements.txt
- python -m pytest -q
- git add .
- git commit -m "..."
